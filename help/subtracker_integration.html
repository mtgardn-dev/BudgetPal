Read Contract (Subscriptions)

BudgetPal reads active subscriptions from SubTracker using the required view:

v_budgetpal_subscriptions

BudgetPal validates SubTracker view contract version at startup and when syncing.

Refresh Subscriptions Flow

  1. User selects Year/Month on Bills tab.
  2. User clicks Refresh Subscriptions.
  3. BudgetPal pulls active SubTracker subscriptions.
  4. BudgetPal creates/updates month bill occurrence rows sourced from SubTracker.
  5. Category mapping is done by category ID contract; fallback is Uncategorized with warning logs when IDs mismatch.

Write Contract (Payments)

Subscription payment posting is handled in the Send Subscription Payments dialog. The dialog lists subscription-flagged expense transactions for the selected month.

StepDescription
Select rowsUse send checkboxes or Select All/Reset All.
Map subscriptionChoose subscription name+ID per row.
Adjust amount (optional)Edit amount directly if needed before posting.
Process PaymentsBudgetPal posts or updates SubTracker payment records using idempotent logic.

Failure Modes

  • Missing/invalid SubTracker DB path in Settings.
  • Missing view or version mismatch in SubTracker schema.
  • Category ID mismatch between systems (falls back to Uncategorized with logs).
