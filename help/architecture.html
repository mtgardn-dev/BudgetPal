Layered Structure

LayerResponsibility
UI (Qt6/PySide6)Tabs, dialogs, table models, user interactions.
ServicesBusiness logic (transactions, transfers, bills, income, budgeting, reconciliation, reporting).
RepositoriesSQLite persistence and query operations.
Persistence CoreDB connection, schema creation, migrations.
ImportersXLSX parsing/validation and SubTracker integration flow.

Core Domain Areas

  • Transactions: expense/income/transfer rows, notes, flags, source metadata.
  • Transfers: manual and rule-generated transfer pairs with transfer-group identifiers.
  • Accounts: institution, alias, account class, account number, line of credit, flags.
  • Month Settings: beginning balances and statement reconciliation values per account per month.
  • Bills / Income / Budget Allocations: global definitions and month instances.
  • Categories: expense vs income type catalog used across the app.
  • Reporting: static report catalog + common DOCX template pipeline.

Path Resolution and Runtime Mode

  • BudgetPalPathRegistry is the single path authority.
  • Development mode resolves paths from project structure.
  • Bundled mode resolves immutable resources from bundle and writable data from user app-data locations.
  • Help assets, logo, templates, and config are all path-registry driven.

Reporting Architecture

  • Reports are defined in static catalog metadata (core/report_catalog.py).
  • Main window maps report keys to report builders.
  • All exported reports use a shared DOCX brand/header template.
  • Report outputs are generated in batch from selected rows on Reports tab.

Build and Packaging

  • scripts/app_build.sh: app bundle build with embedded resources and version metadata.
  • scripts/app_make_dmg.sh: DMG packaging.
  • scripts/release_macos.sh: orchestrates build + DMG with version argument/prompt.
