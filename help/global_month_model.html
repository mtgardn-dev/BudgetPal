Global vs Month Instances

How definitions, refresh, and month edits interact

Concept

BudgetPal separates reusable templates (global definitions) from period-specific editable data (month instances). This keeps recurring setup stable while allowing month-by-month reality to differ.

Global Definitions

  • Recurring Bills (global bill templates)
  • Recurring Income (global income templates)
  • Budget Allocations (global category allocation defaults)
  • Accounts (institution/account metadata and flags)
  • Transfer Rules (rule-based transfer generation mappings)
  • Categories (expense/income classification catalog)

Month Instances

  • Bills tab: bill occurrences for selected Year/Month.
  • Income tab: income occurrences for selected Year/Month.
  • Budget Allocations tab: allocation lines for selected Year/Month.
  • Transactions/Transfers: month activity rows, plus optional prior uncleared rows in account reconciliation contexts.
  • Accounts tab settings by month: beginning balance and statement reconciliation values.

Refresh Actions (Overwrite Model)

ActionWhat It RebuildsOverwrite Risk
Refresh BillsMonthly bill occurrences from global recurring bill definitions.Month edits to bill instances may be replaced.
Refresh SubscriptionsSubscription-derived month bill occurrences from SubTracker.Month edits to subscription-derived rows may be replaced.
Refresh IncomeMonthly income occurrences from global income definitions.Month edits to income instances may be replaced.
Refresh Budget CategoriesMonthly budget allocation rows from global budget definitions.Month edits to allocation instances may be replaced.

Run refresh actions deliberately and only after confirming the selected Year/Month.

Transaction Import Replacement Scope

XLSX imports are tagged by an import period key (Year/Month) and replace spreadsheet-sourced rows for that same period key on re-import. This preserves the spreadsheet-as-source-of-truth workflow while retaining prior-month history.

Transfer Rules and Month Data

  • Transfer rules are global definitions stored in Settings.
  • Rule-based transfers are generated during import from matching transaction rows.
  • Manual transfers are managed on Transfers tab and retained separately from rule-generated transfer rows.
  • Re-import removes prior rule-generated transfers for the import period and regenerates from current rules/input rows.
